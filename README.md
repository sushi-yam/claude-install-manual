# Claude Code インストールマニュアル ― 「書斎をひとつ建てる」課題

Windows の自分のパソコンに、AI と一緒に作業できる **書斎** を建てるための導入手順書を作る課題です。

成果物は2つ:

- `claude-install-manual.md` ― 初心者(プログラミング未経験のクラスメイト)が、迷わず Claude Code と agent-logs をセットアップできるところまで連れて行く本編
- この `README.md` ― なぜこのマニュアルなのか/誰のためなのか/どんな例えで世界観をそろえたのかを示す設計メモ

## このマニュアルは誰のために書かれているか(VPC ― Value Proposition Canvas)

### Customer Profile(読み手 ― web3概論を取る学生)

#### Jobs(やりたいこと)
- web3 概論の課題を、ちゃんと提出する
- 「AIと一緒にコードを書く」を初めて体験する
- 自分の作品を世界に出せる足場(本棚 → ショーケース)を持つ
- 「プログラマーになる」ことではなく、**創作者として技術を道具にする**

#### Pains(痛み・不安)
- **黒い画面(ターミナル)が怖い** ― クリックではなくキーボードで命令する世界に踏み込めない
- **横文字の呪文が意味不明** ― WSL、OAuth、sudo、bash…意味が分からないまま打ち込むことへの抵抗
- **インストールで時間が溶ける** ― 設定で力尽きて、課題本体に着手できない
- **エラーが出ると何が悪いか分からない** ― 真っ赤な英文に心が折れる
- **「私はプログラマーじゃない」という思い込み** ― 自分は対象外だと感じてしまう
- **一度壊したら全部やり直しになる気がする** ― 戻るボタンが見えない不安

#### Gains(得たいもの)
- 「できた」の手触り(`> hello` に Claude が返事をくれた瞬間)
- AIと協業した実感 ― これは自分専属の助手だ、という感覚
- 課題提出の効率化(同じ環境を毎回ゼロから作らなくていい)
- 一度建てれば住み続けられる、自分の書斎
- クラスメイトに「いつの間に?」と言わせる、ささやかな越境感

### Value Map(このマニュアルが提供するもの)

#### Pain Relievers(痛みをやわらげる仕掛け)
- **すべての操作に「例え」がついている** ― 抽象語を、家・机・本棚など身体感覚に翻訳する
- **各ステップに「成功サイン」と「つまずきポイント」を明示** ― 進んでいるのか迷子なのかが、自分で判定できる
- **「よくあるトラブル」で先回り** ― 真っ赤なエラー文の前に、対処手順が用意されている
- **所要時間(30〜60分)を冒頭で明示** ― 「これは長期戦か?」の不安を最初に消す
- **コマンドはコピペ前提** ― 一文字も手で打たなくていい

#### Gain Creators(達成感を増幅する仕掛け)
- **書斎メタファーで「自分の創作環境を建てている」体験に変換** ― 単なるインストール作業ではなくなる
- **完了直後に `> hello` を試させる** ― AIから返事が返る瞬間を、ゴールテープに設計
- **agent-logs を「研究ノート」として位置付け** ― Chiba Tech / Henkaku Center のリサーチプログラム参加者として、自分の作業に意味があると感じられる

## 例えのマップ(世界観の整合性)

このマニュアル全体で、ひとつの統一された比喩世界 ― **「書斎を構える」** ― を使います。すべての例えはこの世界観の中で互いに矛盾しないように設計しました。

| 技術概念 | 例え | 役割 |
|---|---|---|
| Windows | あなたの家 | すでに住んでいる場所 |
| WSL2 | 家の中に増築する **書斎** | Linux を動かすための専用空間 |
| Ubuntu | 書斎の内装(壁紙・床材) | Linux ディストリビューション |
| ターミナル | 書斎の **作業机** | コマンドを打ち込む場所 |
| bash / zsh | 机での **作業ルール(方言)** | シェルの違い |
| `~/.bashrc` | 机に座る前に開く **備忘録** | シェル設定ファイル |
| Antigravity | AI助手つきの **執筆机ツール** | Google製IDE |
| **Claude Code** | 専属の **共同作業者(司書兼編集者)** | AIコーディングアシスタント本体 |
| **agent-logs** | **研究ノート** | セッションログ共有ツール |
| `curl ... \| bash` | **遠方の師匠が直送する道具一式を、その場で組み立てる** | ワンライナーインストール |
| `sudo` | 家の管理者の **印鑑** | 管理者権限 |
| **GitHub** | **本棚** | 完成稿・草稿を整理して置く場所 |
| **Vercel** | **ショーケース** | 世界に向けて本を並べる場所 |

### 流れの一貫性

書斎(WSL)で作業 → 共同作業者(Claude Code)と原稿を書く → 本棚(GitHub)に納める → ショーケース(Vercel)に並べる

この一本の動線が、入門から公開まで貫きます。week03 の範囲は最初の2つ ― **書斎を建てて、共同作業者を迎える** ― までです。

## なぜ web3概論の課題に?

web3 が掲げる「自律した個人が、自分の表現と道具を持つ」という思想は、**まず自分の創作環境を自分の手で建てられること** から始まると考えています。

クラウドの誰かのサービスに乗るのではなく、自分のパソコンの中に書斎を建て、AIという同僚を雇い、本棚に作品を納め、ショーケースに並べる ― この一連の所作を、最初の一歩(書斎を建てる)から言葉にしてみたかった、というのが動機です。

「呪文を覚える」ではなく、「家を建てる」感覚で、最初のターミナルに触れてもらえるように設計しました。
